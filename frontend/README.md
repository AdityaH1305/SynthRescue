# 🖥️ SynthRescue - Tactical Command Interface

This is the frontend client for **SynthRescue**, designed as a high-urgency, military-grade operational dashboard for real-time disaster triage. 

Built to eliminate visual clutter during crisis events, this interface allows first responders to upload drone imagery, visualize live YOLO bounding box telemetry, and read real-time emergency dispatch reports generated by Google Gemini.

## 🚀 Tech Stack

* **Framework:** Next.js 15 (App Router)
* **Language:** TypeScript
* **Styling:** Tailwind CSS v4 (Custom Dark Mode, Glassmorphism, Tactical Grid)
* **Animations:** Framer Motion (Typewriter effects, radar sweeps, severity pulses)
* **Icons:** Lucide React

## ⚙️ Local Development Setup

### 1. Install Dependencies
Ensure you are inside the `frontend` directory, then install the required packages:

```bash
npm install
```

### 2. Configure Environment Variables
Create a `.env.local` file in the root of the `frontend` folder. This tells the frontend where to send the uploaded images for AI processing.

```env
# For local testing, point this to your FastAPI server
NEXT_PUBLIC_API_URL=http://localhost:8000

# For production, replace this with your Google Cloud Run URL
# NEXT_PUBLIC_API_URL=[https://synthrescue-backend-xyz.a.run.app](https://synthrescue-backend-xyz.a.run.app)
```

### 3. Run the Development Server
Start the Next.js local server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to view the tactical dashboard. 

*(Note: Ensure your Python FastAPI backend from the `/backend` folder is running simultaneously so the "Initiate Scan" functionality works).*

## ☁️ Deployment

This frontend is optimized for zero-config deployment on **Vercel**. 

When deploying to production, simply link your GitHub repository to Vercel, set the Framework Preset to `Next.js`, and ensure you add `NEXT_PUBLIC_API_URL` (pointing to your live Cloud Run backend) to your Vercel Environment Variables before triggering the build.
